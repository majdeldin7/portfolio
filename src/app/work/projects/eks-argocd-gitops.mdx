## Overview

In this project, We architected and deployed a highly available, production-grade Kubernetes environment using Amazon Elastic Kubernetes Service (EKS). To streamline the deployment process and enforce infrastructure-as-code principles, We integrated ArgoCD to establish a robust GitOps workflow.

## Architecture & Tech Stack

- **Cloud Provider:** AWS
- **Infrastructure as Code:** Terraform
- **Container Orchestration:** Amazon EKS
- **Continuous Delivery:** ArgoCD (GitOps)
- **Version Control:** GitHub / Git

## The Challenge

The engineering team was facing inconsistencies between environments and manual deployment bottlenecks. We needed a reliable, automated way to deploy microservices that ensured the production state always exactly matched the code repository, while also achieving zero-downtime during application updates.

## The Solution

1. **Automated Infrastructure:** I utilized Terraform to provision the entire underlying infrastructure, including the VPC, subnets, IAM roles, security groups, and the EKS cluster itself, ensuring completely reproducible environments.
2. **GitOps Implementation:** I installed and configured ArgoCD within the EKS cluster. ArgoCD is set up to continuously monitor our Kubernetes manifest repositories and automatically sync any approved changes directly to the cluster.
3. **Zero-Downtime Deployments:** By leveraging Kubernetes rolling update strategies combined with ArgoCD's automated sync mechanisms, I ensured that applications are updated seamlessly without any disruption to the end user.

## Results & Impact

- **100% Automated Deployments:** Completely eliminated manual `kubectl apply` commands and the associated human error.
- **Enhanced Security & Auditability:** Because the Git repository is the single source of truth, every cluster change is now tracked, reviewed, and audited via standard Git pull requests.
- **Faster Disaster Recovery:** In the event of a failure or bad deployment, the cluster state can be instantly and reliably rolled back to a previous known-good Git commit.
